Python Pattern Comparisons

Anti-patterns vs optimized solutions

Dictionary Dispatch
HTTP Status Code Mapping
Anti-Pattern
def get_http_message(status_code): if status_code == 200: return "OK" elif status_code == 201: return "Created" elif status_code == 400: return "Bad Request" elif status_code == 401: return "Unauthorized" elif status_code == 403: return "Forbidden" elif status_code == 404: return "Not Found" elif status_code == 500: return "Internal Server Error" else: return "Unknown" # Worst case: 8 comparisons for status 500
O(n) lookup with sequential comparisons. Performance degrades linearly with more status codes.
O(n) Complexity
Better Solution
HTTP_MESSAGES = { 200: "OK", 201: "Created", 400: "Bad Request", 401: "Unauthorized", 403: "Forbidden", 404: "Not Found", 500: "Internal Server Error", } def get_http_message(status_code): return HTTP_MESSAGES.get(status_code, "Unknown") # Always 1 lookup regardless of status code
O(1) hash table lookup. Constant-time performance regardless of dictionary size.
O(1) Complexity
Function Dispatch Pattern
Calculator Operations
Anti-Pattern
OPERATIONS = {'+': 'add', '-': 'subtract', '*': 'multiply', '/': 'divide'} # Maps to operation names, not behaviors
Mapping values to strings requires additional logic to execute operations.
Indirection
Better Solution
def calculate(x, operator, y): operation = OPERATIONS.get(operator) if operation is None: raise ValueError(f"Unknown operator: {operator}") return operation(x, y) # Map values to functions for behavior selection
Direct function dispatch eliminates intermediate steps. Functions as first-class values.
First-Class Functions
Lazy Evaluation Pattern
Deferred Execution with Lambdas
Anti-Pattern
def process(action): handlers = { 'save': expensive_save(), 'load': expensive_load(), } return handlers.get(action, lambda: None) # WRONG: Called immediately! # Both expensive_save() and expensive_load() # execute regardless of selected action
All functions execute upfront when building the dictionary, wasting computation.
Eager Evaluation
Better Solution
def process(action): handlers = { 'save': lambda: expensive_save(), 'load': lambda: expensive_load(), } handler = handlers.get(action, lambda: None) return handler() # Lazy: called only if selected action matches # Execute the selected function
Lambdas defer execution. Only the selected handler runs, avoiding unnecessary work.
Lazy Evaluation
Usage Examples
Dictionary Dispatch in Practice
Without Dispatch
result = calculate(10, '+', 5) # Returns: 15 result = calculate(10, '/', 2) # Returns: 5.0
Traditional if/elif chains for operation selection. Linear lookup time.
Linear Search
With Dispatch
dispatch_result = calculate(10, '+', 5) # Returns: 15 dispatch_result = calculate(10, '/', 2) # Returns: 5.0
Hash table lookup with function dispatch. Constant-time operation selection.
O(1) Dispatch
